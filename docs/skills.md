# 技能資料庫與職業設定

> 與區域開發的關聯：技能決定 NPC 能力、掉落秘笈、任務獎勵與訓練場內容。此處依 `docs/data/skills.json` 產生的分類統計與代表性條目，方便規劃新區域時快速查找需要的技能屬性。

## 類別統計

| 類別 | 檔案數 | 說明 |
| --- | --- | --- |
| 武器技能 | 11 | 劍、刀、弓、槍、棍、斧、鞭、扇、短兵、拳法、氣功。 |
| 法術技能 | 10 | 火、風、光、聖、雷、水、土、暗、邪、毒系。 |
| 職業技能 | 7 | 格鬥、暗殺、法師、鑄造、吟唱、醫療、盜賊。 |
| 其他技能 | 3 | 步法、技能總覽、技能熟練度。 |

- `scripts/build_docs.py` 會根據 `skill/index.html` 解析分類；若新增類別，記得更新該 HTML 後重跑腳本。
- `skill/learnlv.html` 屬於「技能熟練度」參照頁，不是一般技能明細頁；它應計入 `skill/index.html` 的分類統計，但不應和 `skill/sword.html`、`skill/step.html` 這類有完整 `英文名稱 / 中文名稱 / 領悟技能` 區塊的技能頁等量看待。
- 依 `merc-area-builder` 建議，可將掉落/訓練表格寫入區域 `res` 檔，並引用此表格中的中文/英文名稱避免錯字。

## 區域設計建議

1. **掉落／習得來源**：於 `res` 檔內安排 NPC 擁有特定技能，並在 `mob` 描述中註記「可教導」或「僅領悟」。
2. **熟練度上限**：技能頁面常以「馬馬虎虎」、「神乎其技」描述各職業上限，請在 NPC 對話或任務條件中引用這些詞彙，避免與英文 Rank 混用。
3. **資源消耗**：資料欄位中的「浪費數值」= 體力/法力/氣力，可直接映射成戰鬥節奏；例如步法技能多耗「體力」，在地圖 `SECT_MOUNTAIN` 內可設定較多休息點。
4. **領悟機率**：若 `領悟機率` = 極低，建議搭配 `Help` 或 NPC 提示（如 `realm` 功能區 NPC）提醒玩家機率。
5. **秘笈／study**：對 `merc-area-builder` 來說，任何 `study` 互動都需在區域 `obj` 檔設計 `book` 類物件並於 `help`/`notes` 內說明。

## 與公告／國家系統的連動

- 請對照 `system.md` 的公告時間線：例如 2002/01/25 開放「百幻身法」，若在區域中提供該步法，需標註「2002/01/25 公告後加入」。
- 國家系統指令如 `realm !join` 會查 `技能` 欄位（權限），故 `realm.md` 的表格應與此檔案交叉檢查，避免 NPC 教出國家限定技能。
